## 1. 机器人功能（6 项）

| # | 功能 | 需求 | 干什么 | 飞书能力 |
|---|---|---|---|---|
| 1 | **会议沉淀** | ① | 会议结束→抓妙记纪要（AI总结/决策/待办/逐字稿）→追加到对应项目文档 | `event consume minutes.minute.generated_v1` + `vc +notes` + `docs +update` |
| 2 | **群聊沉淀** | ② | 读群消息→LLM 抽「决策/结论/待办/FAQ」（丢闲聊）→更新对应群沉淀页 | `im +chat-messages-list`/事件 + LLM + `docs +update` |
| 3 | **待办维护** | ③ | 把①②抽到的 action item→建/更新**原生 Task**（负责人/截止/提醒）+ 回链文档 | `lark-task`（create/patch/assign/reminder） |
| 4 | **战略目标维护** | ③ | 把①②里与目标相关的进展→写成**原生 OKR 进展记录**；文档放可读汇总 | `lark-okr`（progress-create/update） |
| 5 | **智能路由 + 幂等**（自研核心） | 支撑①②③ | 决定「写到哪个文档/任务清单/OKR」+ 去重 upsert | **多维表格 Base** 做映射+幂等表 |
| 6 | **周期汇总**（可选·可缓做） | 延伸 | 每周把散落决策/待办/OKR进展自动汇总成「周报」页 | 定时 + `docs +update` |

> **工作量重心是 ⑤**：①②④⑥ 有现成轮子；不做幂等就会重复写、写错文档。

## 2. 公司知识库文件结构（飞书 Wiki 知识空间为骨架）

主轴**按项目组织**；全局索引页只放自动生成的链接，避免重复存储（单一真身 + 多视图）。

```
📁 公司知识库（飞书 Wiki 知识空间）
├── 📌 公司总览
│    ├── 公司战略与年度目标        ← 机器人按周镜像 OKR（功能④）
│    ├── 重要决策记录 Decision Log  ← ①②里"拍板"决策自动汇总
│    └── 组织架构与成员            ← 人工
├── 🗂️ 项目空间（每项目一子页）★主轴
│    └── 项目X
│        ├── 概述 / 需求 / 设计     ← 人工为主
│        ├── 会议纪要              ← 功能① 自动追加
│        ├── 群聊沉淀              ← 功能② 自动更新
│        └── 待办（汇总视图）        ← 功能③ 镜像原生 Task（只读）
├── 📝 会议纪要库（按时间索引）      ← 自动：指向各项目纪要的链接列表
├── 💬 群知识沉淀（按群索引）        ← 自动
├── ✅ 公司待办看板               ← 真身=原生 Task；文档=只读汇总
├── 🎯 OKR / 战略目标             ← 真身=原生 OKR；文档=可读汇总
└── 📚 制度 / SOP / FAQ          ← 人工为主，机器人补充
```

**不在 Wiki、但属于系统的"文件"：**
- **路由映射 + 幂等表**（Base 一张表，见 §4）——机器人写前查它去重。
- （可选）原始素材冷存：逐字稿 / 原始群消息归档。

## 4. 核心：路由 + 幂等（Base 表设计）

一张多维表格，记录"每条外部源 → 写到了哪、写了什么指纹"，写前先查：

| 字段 | 含义 |
|---|---|
| `source_type` | meeting / chat / manual |
| `source_id` | 妙记 minute_token / 消息 message_id / 群+时间窗 |
| `source_meta` | 会议主题 / 群名 / 时间 |
| `target_kind` | doc / task / okr |
| `target_id` | docx document_id / wiki node_token / task guid / okr objective_id |
| `target_block_id` | 写入的块 ID（用于 upsert/覆盖，避免重复块） |
| `content_hash` | 内容指纹（判断是否需更新） |
| `status` | written / updated / skipped |
| `last_synced_at` | 最近同步时间 |

**幂等逻辑**：来一条源 → 按 `source_id` 查表 → 不存在则写入并记录 `target_block_id`+`content_hash`；已存在且 `content_hash` 变了则按 `target_block_id` 覆盖更新；没变则 skip。

## 5. 需求③ 设计：原生 Task/OKR 为真身 + 文档汇总视图（已确认方案 C）

- **待办**：抽到 action item → `lark-task +create`（带负责人/截止/提醒），任务里回链来源（会议/消息 url）；项目页"待办汇总"与"公司待办看板"是**只读镜像**（机器人按 `+get-my-tasks`/`tasklist tasks` 渲染），勾选/改在飞书 Task App 里做。
- **战略目标**：不自动新建 Objective（战略由人定）；机器人只**写进展**——识别"与某 KR 相关的进展"→ `lark-okr +progress-create` 挂到对应 KR；"公司战略与年度目标"页按周 `+cycle-detail` 镜像成可读汇总。
- 单一事实源在原生 Task/OKR，文档只读，避免两处不一致。

## 6. 总体架构（分层）与触发模型

沿用上层文档七层（触发/采集/处理/路由/写入/知识库/问答），本设计落到①-⑥：

```
事件层  event consume（长连接·免公网·内置验签）
  ├─ minutes.minute.generated_v1 → 会议线
  └─ im.message.receive_v1        → 群聊线（+定时兜底拉历史）
采集层  vc +notes / im +chat-messages-list
处理层  妙记 AI 总结（会议）/ 外接 LLM（群聊抽取，必经）
路由层  Base 映射+幂等（§4）——自研核心
写入层  docs +update（文档）/ lark-task / lark-okr / wiki +node-*
知识库  飞书 Wiki（主线·零迁移）
```

触发优先长连接事件；群全量读取受权限限制时退化为"定时拉历史 + @机器人主动归档"。

## 7. 部署模型（关键纠偏）

**机器人必须是独立常驻进程，不绑任何聊天会话**（不依赖 Claude 会话开着）：

- 形态（任选其一）：`lark-cli event consume` 守护进程 / oapi-sdk WebSocket 服务 / OpenClaw·Hermes agent 运行时（`lark-cli config bind`）/ n8n Lark Trigger。
- **令牌**：持久化在 `~/.lark-cli/config.json`，含 `offline_access` 自动刷新。⚠️ 刷新令牌 7 天有效——**进程持续运行就会自动续期**；若超 7 天完全无调用，需重新 `auth login`。
- **身份**：无人值守优先 **bot 身份**（无需 user 令牌续期、不冒充用户）；仅"读个人单聊/个人云盘/个人日历"才需 user 身份。读群全量消息：bot 需入群 + 敏感 scope `im:message.group_msg`（管理员审批），或用 user 身份读 user 可见的群。

## 8. 新公司组织迁移清单（换租户 = 换应用 + 重新授权）

自建应用绑定单一租户，新组织无法平移现有授权。新组织里一次性做：

1. 新企业开发者后台建自建应用 → 新 appId/appSecret，开通所需 scope（§9）；
2. `lark-cli config init --new` 指向新应用（配置按 appId 分 profile，旧配置不删、可共存）；
3. `lark-cli auth login --scope "<§9 全部 scope 一次性列齐>"`（**一次授权到位**，别增量撞墙）；
4. `lark-cli config bind --identity bot-only`（无人值守优先 bot；要 user 资源再 user-default）。

> 建议：生产应用**直接在新组织建**，当前租户这套授权当调研/原型用。

## 10. 风险与前置条件

| # | 风险/前置 | 应对 |
|---|---|---|
| 1 | 群全量消息读取是敏感权限 | 立项前找管理员确认；批不下来退化为"@机器人主动归档" |
| 2 | 妙记 AI 总结依赖企业套餐 | 确认套餐；无 AI 总结则外接 LLM 兜底 |
| 3 | 写目标文档需机器人是协作者且有编辑权 | 机器人建的文档天然有权；写存量文档先加协作者 |
| 4 | 幂等/去重 | §4 Base 表做 upsert |
| 5 | 群消息信噪比 | 时间窗批量 + LLM 只抽有价值内容 + 写前去重 |
| 6 | 令牌 7 天刷新窗口 | 常驻进程持续运行自动续期；user 身份才有此问题，bot 身份无 |
| 7 | 新组织迁移 | §8 清单 |
