# 阶段 3：综合分析

收集所有研究员的报告，合成一个完整的分析：

0. **需求完整性验证（MECE）**：逐条对照用户原始需求，检查每个需求点是否都有对应的研究结论。特别注意：
   - 用户说了 N 个需求，研究结论是否覆盖了全部 N 个？
   - 开放式需求（"等等"）是否已被代码分析员展开为具体列表？
   - 展开后的列表是否需要**向用户确认范围**再继续？
   - 如果有遗漏，**停下来补充研究**，不要带着缺口进入方案设计
1. **交叉验证**：UX 建议的方案，技术上可行吗？技术建议的方案，UX 上合理吗？
2. **冲突解决**：如果不同维度的建议冲突，明确说出冲突点，优先 UX（用户体验 > 技术优雅）
3. **范围确认**：改动范围是否可控？会不会引入新问题？

**必须输出给用户**（不允许跳过展示）：
- "{N} 个问题的根因是 {X}"
- "UX 分析发现 {Y}，建议改为 {Z}"
- "技术上推荐方案 {A}，因为 {reason}"
- "涉及 {M} 个文件"
- `--- 阶段 3 完成 ---`
