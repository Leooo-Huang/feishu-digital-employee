# 步骤 1：扫描设计文档

读取 `docs/plans/` 下的所有设计文档，提取可追踪的条目：

**从 `*-ideation.md` 提取（按层级分组追踪）：**
- **平台层功能**（P 编号）：每个平台能力是一个追踪项，追踪标签带 `[P]` 前缀
- **业务场景层功能**（W 编号）：每个业务场景是一个追踪项，追踪标签带 `[W]` 前缀
- V2 功能列表（保留 P/W 标签）
- Future 功能列表（保留 P/W 标签）

**分层追踪规则：** 平台层功能的完成是场景层功能可用的前提。如果一个场景层功能依赖的平台层功能未完成，即使场景层代码存在，也应标为 `[~]` 并注明"依赖 P{n} 未就绪"。

**从 `*-design.md` 提取：**
- 架构组件列表
- 技术选型决策

**从 `*-ui.md` 提取：**
- 页面清单（每个页面是一个追踪项）
- 用户流程列表

**从 `*-api.md` 提取：**
- 数据模型列表（每个实体是一个追踪项）
- API 端点列表（每个端点是一个追踪项）

**从 `*-plan.md` 提取（Harness Engineering 对齐）：**
- 任务列表（每个 Task 是一个追踪项）
- **`acceptance_criteria`**：如果 task 有 acceptance_criteria 字段，提取每条标准作为子追踪项
- **`status`**：读取 task 的 status 字段（pending / completed），与代码实际状态对比
- **`decision`**：如果 task 有 decision 字段，记录偏离决策（同步时保留，不覆盖）
- **`review_result`**：如果 task 有 review_result 字段（GAN 审查分数），记录并保留

**从 `*-index.md` 提取：**
- 知识地图条目（每行指针是否指向实际存在的文件和章节）
- 如果发现指针指向不存在的文件或章节 → 标记为 `[断链]`

**从 `*-rules.md` 提取：**
- 技术栈版本号 → 与 package.json/pyproject.toml 实际版本对比
- 如果发现版本不一致 → 标记为 `[版本漂移]`
