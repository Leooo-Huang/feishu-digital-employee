# 场景依赖图检查清单

在场景层功能排序完成后，绘制场景间的数据流依赖图：

## 格式

```
W{n} → W{m}：{W{n}的什么输出} 是 {W{m}的什么输入}
独立：W{x}（不依赖其他场景的输出，也不产出其他场景需要的输入）
```

## 示例

```
W1(市场调研) → W2(营销工厂)：调研报告中的风格方向和爆款数据 是 营销工厂的素材生成输入
独立：W3(客服工单)、W4(产品上架)
```

## 检查规则

- 如果 A 的输出是 B 的输入，B 的 W 编号必须大于 A（编号反映工作流顺序，不是优先级排序）
- W 编号 = 用户使用产品时的自然顺序（先做什么后做什么），不是功能重要性排序
- 如果存在循环依赖（A→B→A），说明场景拆分有问题，需要重新拆分
