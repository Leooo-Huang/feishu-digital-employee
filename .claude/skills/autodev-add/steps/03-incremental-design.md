# 步骤 3：增量技术设计

**目标**：设计新功能的技术方案，融入现有架构，不破坏已有设计。

## 执行

1. **评估架构影响**：
   - 新功能能用现有技术栈实现吗？还是需要引入新依赖？
   - 需要新建组件/模块吗？还是扩展现有组件？
   - 对现有数据模型有影响吗？需要加表/加字段？

2. **设计新功能的架构**：
   - 如果**融入现有模块**：描述在哪个组件中添加什么
   - 如果**新建模块**：描述新模块的职责、接口、与现有模块的交互

3. **技术鲜度验证**（如果引入新依赖）：
   - WebSearch 验证新依赖的最新稳定版本
   - 确认与现有技术栈的兼容性

4. **检查能力-组件映射**：
   - 新功能对应的能力，由哪个组件主责？
   - 更新映射表

## 追加到文档

在 `*-design.md` 中追加新章节：
- "## 新增功能：{名称}"
- 包含：架构影响、新组件/扩展描述、新依赖（如有）、更新的映射表

**规则**：只追加，不修改已有章节（除非新功能确实需要改动现有架构，此时必须标注修改原因）。
