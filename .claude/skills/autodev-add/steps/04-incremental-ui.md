# 步骤 4：增量 UI 设计

**目标**：设计新功能的 UI/UX，与现有页面风格和导航保持一致。

## 执行

1. **读取现有 UI 设计**：
   - `*-ui.md` — 现有页面清单、信息架构、导航模式
   - **视觉规范章节** — 品牌色、排版、圆角、间距、阴影（确保新功能匹配）
   - **动效规范章节** — 动效程度、过渡参数、场景方案（确保新功能风格一致）
   - 理解现有的设计语言和交互模式

2. **确定新功能的 UI 形态**：
   - **新页面**？→ 放在导航的什么位置？
   - **扩展现有页面**？→ 加在哪个区域？会不会信息过载？
   - **弹窗/面板**？→ 从哪个入口触发？

3. **设计新功能的页面**：
   - 按现有 ui.md 的格式，为新页面/组件撰写：
     - 布局区域划分
     - 组件清单
     - 状态设计（加载/空/正常/错误）
     - 用户流程

4. **视觉一致性检查**：
   - 新页面/组件使用的颜色是否来自现有色彩系统？（不引入色板外的颜色）
   - 字体层级是否匹配现有排版规范？
   - 间距和圆角是否使用现有刻度？
   - 动效时长和缓动函数是否与现有页面一致？
   - 参考 `autodev-ui/checklists/ui-completeness.md`

5. **导航一致性检查**：
   - 新页面在信息架构中的位置符合用户心智模型吗？

## 追加到文档

在 `*-ui.md` 中追加新章节：
- 信息架构更新（如有新的导航项）
- 新页面/组件的完整设计
- 新增的用户流程
