# 步骤 7：更新项目规格摘要

**目标**：把新功能的信息同步到 index.md 和 rules.md。

## 执行

1. **更新 `*-index.md`**：
   - MVP 功能清单追加新功能条目
   - 能力-组件映射表追加新映射（如有）
   - 关键设计决策追加（如引入了新技术/新依赖）

2. **更新 `*-rules.md`**（如果新功能引入了新约束）：
   - 新依赖的版本号追加到技术栈约束
   - 新 API 端点的约定（如果和现有有差异）
   - 新的禁止模式（如果新功能有特殊限制）

3. **验证更新后的行数**：
   - index.md 仍然 < 100 行？如果超出，删除低优先级信息
   - rules.md 仍然 < 80 行？如果超出，合并相似规则
