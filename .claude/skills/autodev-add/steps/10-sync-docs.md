# 步骤 10：同步文档

**目标**：更新所有设计文档的完成状态。

## 执行

1. **调用 `/autodev-sync`**：
   - 自动扫描代码，更新各文档中新功能的状态标记
   - 检测设计漂移（新功能实现是否偏离了步骤 2-5 的设计）

2. **如果实现偏离了设计**：
   - 修正对应文档中的设计描述
   - 标注修正原因和日期
   - 更新 index.md/rules.md（如受影响）

3. **向用户汇报**：
   - "新功能 {名称} 已完成"
   - "涉及文件：{列表}"
   - "设计文档已更新：{列表}"
   - 如有回归风险："建议关注 {具体区域}"
