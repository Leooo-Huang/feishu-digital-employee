# 步骤 5：增量 API 设计

**目标**：设计新功能需要的 API 端点和数据模型变更，复用现有的 API 约定。

## 执行

1. **读取现有 API 设计**：
   - `*-api.md` — 现有数据模型、端点清单、API 约定
   - 理解现有的认证方式、错误格式、分页规范

2. **数据模型变更**：
   - 新功能需要新表/新实体吗？
   - 需要在已有实体上加字段吗？（标注对现有数据的影响）
   - 新实体与现有实体的关系（外键、引用）

3. **新增端点设计**：
   - 从 UI 数据需求倒推：新页面需要什么数据？
   - 按现有 api.md 的格式设计每个端点：方法、路径、请求体、响应体、错误码、业务规则
   - 遵循现有 API 约定（命名风格、认证方式、分页格式）

4. **回归影响评估**：
   - 新增的数据模型变更会影响现有端点吗？
   - 现有端点的返回格式需要变吗？

## 追加到文档

在 `*-api.md` 中追加：
- 新增/修改的数据模型
- 新端点清单和详细设计
- 标注对现有端点的影响（如有）
