# Evaluator 评分标准

> "Criteria wording shapes output" — 措辞直接影响模型的收敛方向。
> 本 rubric 的措辞经过设计，每个分数等级的描述会引导 Evaluator 向更高标准收敛。

## 评分维度

### 1. Design System Compliance（设计系统合规）

评什么：颜色、字体、间距、圆角、阴影是否严格匹配 DESIGN.md 的 token。

| 分数 | 标准 |
|------|------|
| **5** | **Pixel-perfect fidelity**：每一个颜色值、字体族、字号、字重、间距、圆角、阴影都能在 DESIGN.md 中找到精确对应。CSS 中零硬编码视觉值——全部通过变量或 token 引用。像是设计系统作者自己写的代码。 |
| **4** | **Near-perfect**：1-2 处次要偏离（如某个 hover 态颜色略有出入），但所有主要 token（主色、字体族、主要间距）完全正确。 |
| **3** | **主要 token 正确但有遗漏**：能看出在使用设计系统，但仍有明显区域用着框架默认样式或硬编码值。 |
| **2** | **部分遵守**：主色可能对了，但字体、间距、阴影大部分是 generic 的。感觉像"试了一下但没做完"。 |
| **1** | **基本未应用**：看不出与 DESIGN.md 有任何关联。 |

**审查方法**：
- Grep 代码中的硬编码色值（`#xxx`, `rgb(`, `hsl(`）— 每个都是扣分点
- 检查 CSS 变量定义是否与 DESIGN.md token 一致
- 对比截图中的实际颜色与 DESIGN.md 规定的颜色

---

### 2. Visual Craft（视觉工艺）

评什么：间距精度、视觉层级、对齐、对比度、留白——技术执行的专业程度。

| 分数 | 标准 |
|------|------|
| **5** | **Professional design studio quality**：间距节奏感强且一致，视觉层级清晰（标题 > 正文 > 辅助文字一眼可辨），所有元素对齐精确，留白舒适。达到 Stripe 官网或 Linear 产品页的水准——一眼看上去就是专业设计师的作品。 |
| **4** | **接近专业**：整体节奏好，1-2 处间距不一致或对齐偏差。不细看发现不了问题。 |
| **3** | **功能性足够但缺乏精致感**：间距基本合理但不够一致，视觉层级存在但不够鲜明。"能用但不好看"。 |
| **2** | **粗糙**：间距明显不一致，元素挤在一起或过于松散，对齐偏差肉眼可见。典型的"开发者写的 CSS"。 |
| **1** | **混乱**：布局崩坏，间距无规律，元素重叠或完全不对齐。 |

**审查方法**：
- 截图中标题 → 正文 → 辅助文字的字号/字重递减是否清晰
- 同级元素之间的间距是否一致
- 卡片内边距是否统一
- 页面整体是否有呼吸感（留白 vs 拥挤）

---

### 3. Coherence（视觉一致性）

评什么：整个页面是否感觉像同一个设计系统出品，而不是拼凑的。

| 分数 | 标准 |
|------|------|
| **5** | **One voice**：页面上的每个组件——从导航栏到按钮到表单到页脚——看起来像出自同一个设计师之手。设计语言完全统一。 |
| **4** | **基本统一**：95% 的组件风格一致，1-2 个组件略有"外来感"（如某个弹窗用了不同的圆角风格）。 |
| **3** | **部分统一**：核心组件风格一致，但边缘组件（toast、tooltip、空状态）明显不属于同一套系统。 |
| **2** | **拼凑感**：多种风格混杂。部分区域是设计系统风格，部分区域是框架默认样式，部分是自定义样式。 |
| **1** | **完全不统一**：每个组件像来自不同的项目。 |

**审查方法**：
- 扫描截图：导航栏、内容区、侧边栏的风格是否一致
- 检查不同组件的圆角、阴影、颜色是否用了同一套 token
- 注意"被遗忘的角落"：加载状态、空状态、错误提示是否也应用了设计系统

---

### 4. Functionality Preservation（功能完整性）

评什么：打磨后所有原有功能是否正常，无回归。

| 分数 | 标准 |
|------|------|
| **5** | **Zero regression**：所有可见功能正常。按钮可点击，表单可提交，导航正常跳转，数据正常显示。控制台无新增错误。 |
| **4** | **极微小回归**：功能全部正常，但有 1 个非关键的视觉瑕疵（如某个 tooltip 位置偏移 2px）。 |
| **3** | **有可见回归**：核心功能正常，但 1-2 个次要功能受影响（如 hover 效果丢失、某个动画不播放）。 |
| **2** | **明显回归**：部分核心功能受影响（按钮不可点击、表单布局错位、数据不显示）。 |
| **1** | **严重回归**：页面无法使用或白屏。 |

**审查方法**：
- 截图中所有按钮/链接是否可见且有交互指示
- 如果有 playwright：尝试点击主要按钮，确认无 JS 错误
- 检查控制台是否有新增的错误/警告

---

## PASS / FAIL 判定

```
总分 ≥ 16/20 且 无单项 ≤ 2 → PASS
Functionality ≤ 2 → 立即 FAIL（功能回归是硬性否决）
其他 → NEEDS_IMPROVEMENT
```

## Evaluator 行为约束

<IMPORTANT>
1. 不要给出"鼓励性"评分。3 分不是"还可以"，是"能用但不好看"。
2. 不要因为"比上一轮好了"就给高分。评分标准是绝对的（对照 DESIGN.md），不是相对的。
3. Compliance 维度必须通过代码审查验证，不能只看截图。截图中颜色可能因渲染差异而误判。
4. 每个扣分必须附带具体原因和位置。"间距不太对"不是有效反馈，"标题与副标题间距 32px，DESIGN.md 规定 24px"才是。
5. 如果不确定某处是否符合 DESIGN.md，判为不符合。宁可误判严格，不可误判宽松。
</IMPORTANT>
