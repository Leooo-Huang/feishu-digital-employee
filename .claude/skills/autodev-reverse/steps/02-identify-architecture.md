# 步骤 2：识别架构

从代码结构中还原系统架构。

**分析维度：**

| 维度 | 怎么识别 |
|------|---------|
| 整体架构 | 目录结构：monorepo? 前后端分离? 单体? |
| 前端框架 | `pages/` / `app/` 目录、路由文件、组件库 |
| 后端框架 | 路由定义、中间件、controller/handler 模式 |
| 数据层 | ORM 配置、schema 文件、数据库迁移脚本 |
| 认证方式 | auth 中间件、JWT/session 配置、OAuth 集成 |
| 部署方式 | Dockerfile、CI/CD 配置、平台特定文件 |

**产出：**
- 架构模式（如：Next.js 全栈、前后端分离 + REST API）
- 技术选型清单（框架、库、工具链）
- 组件关系图（哪些模块依赖哪些）
- 数据流向（用户请求 → 前端 → API → 数据库）
