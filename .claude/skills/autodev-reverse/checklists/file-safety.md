# 文件删除安全规则

本 skill 的职责是**生成和整理文档**，不是清理项目文件。必须严格遵守以下规则：

## 绝对禁止删除的文件类型
- **媒体文件**：视频（.mov/.mp4/.webm/.avi）、图片（.png/.jpg/.jpeg/.gif/.svg）、音频
- **数据文件**：数据库（.db/.sqlite）、数据集、模型文件（.task/.onnx/.pt）
- **配置文件**：.env、.gitignore、lock 文件
- **测试资源**：测试用的媒体文件、fixture 数据、snapshot 文件
- **用户资产**：任何非 .md 文件，除非用户**逐个明确指定**

## 可以建议删除的（仅限 .md 文档）
- 内容已被其他文档 100% 覆盖的 .md 文件
- 描述已废弃架构/技术栈的 .md 文件

## 删除流程
1. **只列出 .md 文档的删除建议**，不要把非文档文件混进来
2. **每个文件单独列出**，不要捆绑成一个选项让用户批量确认
3. **删除前检查 Git 状态**：`git ls-files <file>` 确认文件是否被 Git 跟踪
4. **未被 Git 跟踪的文件**：必须额外警告"此文件不在 Git 中，删除后无法恢复"
5. **使用安全删除**：优先用 `git rm`（已跟踪文件）；未跟踪文件必须让用户自己手动删除，不要用 `rm`
