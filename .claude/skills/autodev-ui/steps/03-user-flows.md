# 步骤 3：用户流程

对每个 MVP 功能，画出完整的用户操作流程。

**格式：**
```
流程：{功能名称}
触发：用户从哪里开始（入口）
1. 用户看到 {什么}
2. 用户操作 {做什么}
3. 系统响应 {发生什么}
4. ...
完成：用户得到 {什么结果}
异常：如果 {X 出错}，用户看到 {什么}
```

**关键要求：**
- 每个流程都要有**入口**和**出口**
- 标注**决策点**（用户需要选择时）
- 标注**等待点**（需要加载/处理时，用户看到什么）
- 覆盖**异常路径**（失败、取消、超时）

<IMPORTANT>
流程必须从用户视角描述："用户看到..."、"用户点击..."，而不是"系统渲染..."、"API 返回..."。
技术细节在 API 设计阶段处理，这里只关心用户体验。
</IMPORTANT>
