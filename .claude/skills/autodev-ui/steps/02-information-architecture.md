# 步骤 2：信息架构

设计整体页面层级和导航关系。

**思考框架：**
- 用户的**主要任务**是什么？-> 这些任务应该最容易到达
- 哪些内容是**频繁访问**的？-> 放在导航层级的顶部
- 哪些内容是**偶尔访问**的？-> 可以放在更深的层级
- 不同用户角色的入口是否不同？

**产出：**
```
首页
├── 功能 A（主要任务）
│   ├── 子页面 A1
│   └── 子页面 A2
├── 功能 B（主要任务）
├── 设置
│   ├── 个人信息
│   └── 偏好设置
└── ...
```

**导航模式选择：**
- 页面少（<5 个顶级）：Tab bar / 侧边栏
- 页面多（>5 个顶级）：汉堡菜单 + 核心功能固定
- 有明确层级：面包屑导航
- 单一主任务：底部导航 + 浮动操作按钮
