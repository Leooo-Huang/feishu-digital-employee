# 步骤 5c：动效规范（L4）

**目标**：定义项目的动效标准，确保交互反馈一致且不过度。

## 3 个核心决策

### 1. 动效程度

选择一个整体基调：

| 程度 | 特征 | 适用场景 |
|------|------|---------|
| **极简** | 仅状态切换有淡入淡出，无复杂动画 | 工具类、数据密集型产品 |
| **适度**（推荐） | 关键操作有反馈动画，页面切换有过渡 | 大部分 SaaS 和消费产品 |
| **丰富** | 列表项逐个入场、拖拽有物理感、大量微交互 | 创意工具、游戏化产品 |

选择：{极简 / 适度 / 丰富}

### 2. 过渡参数

| 类型 | 时长 | 缓动函数 | 用途 |
|------|------|---------|------|
| 即时反馈 | 100-150ms | `ease-out` | hover、focus、toggle |
| 标准过渡 | 200-300ms | `ease-in-out` | 展开/折叠、下拉、Tab 切换 |
| 入场动画 | 300-400ms | `ease-out` | 模态框弹出、抽屉滑入 |
| 退场动画 | 200-250ms | `ease-in` | 模态框关闭、toast 消失 |
| 页面过渡 | 300-500ms | `ease-in-out` | 路由切换（如适用） |

### 3. 具体场景

| 场景 | 方案 | 说明 |
|------|------|------|
| 页面切换 | {无 / 淡入淡出 / 滑动} | 路由级过渡 |
| 数据加载 | {骨架屏 / 旋转器 / 渐进式} | 异步内容占位 |
| 列表更新 | {直接替换 / 淡入淡出 / 布局动画} | 增删改后的列表变化 |
| 按钮反馈 | {无 / 缩放(scale) / 涟漪(ripple)} | 点击反馈 |
| 成功操作 | {无 / 对勾动画 / toast 弹出} | 表单提交、保存成功 |
| 错误反馈 | {无 / 抖动(shake) / 高亮闪烁} | 表单验证失败 |
| 拖拽 | {无 / 跟随光标 + 阴影提升} | 如有拖拽功能 |

## 规则

- 动效不能阻塞用户操作——所有动画期间用户应该能继续交互
- 退场动画应比入场动画短（用户关闭东西时不想等）
- 首屏加载不要有入场动画——用户等了加载已经够久了
- 如果用户系统开启了"减少动态效果"（prefers-reduced-motion），应尊重该设置
- 所有时长用 CSS 变量或常量定义，不硬编码
