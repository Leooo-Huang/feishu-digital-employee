# 步骤 4：页面详细设计

对每个页面/屏幕，定义：

```
页面：{页面名称}
用途：{这个页面解决什么问题}
入口：{用户从哪里来}
出口：{用户去哪里}

布局区域：
- 头部：{导航、标题、操作按钮}
- 主体：{核心内容}
- 侧边：{辅助信息}（如有）
- 底部：{操作栏}（如有）

核心组件：
- {组件名}：{做什么}（交互：{怎么操作}）
- ...

数据需求：
- 这个页面需要展示什么数据
- 数据从哪来（列出需要的数据，不需要写 API，留给 API 设计阶段）
```

**设计原则：**
- 默认状态应该是最简洁的，细节由用户主动触发
- 一个页面一个主要任务，不要堆砌功能
- 信息展示遵循渐进式披露：摘要 -> 详情 -> 完整数据
