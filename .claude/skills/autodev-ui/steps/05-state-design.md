# 步骤 5：状态设计

对每个页面，定义所有可能的状态：

| 状态 | 用户看到什么 | 用户能做什么 |
|------|-------------|-------------|
| **加载中** | 骨架屏/加载指示器 | 等待（或取消） |
| **空状态** | 引导文案 + 行动按钮 | 开始使用 |
| **正常** | 内容 | 正常操作 |
| **错误** | 错误信息 + 重试按钮 | 重试或返回 |
| **部分加载** | 已有内容 + 加载更多 | 浏览已有 + 加载更多 |
| **离线**（如适用） | 缓存内容 + 离线提示 | 浏览缓存 |

<IMPORTANT>
空状态是最容易被忽略但最影响新用户体验的状态。每个列表页、数据页都必须设计空状态：不是空白，而是引导用户创建第一条数据。
</IMPORTANT>
