# UI/UX 设计完整性检查清单

- [ ] 每个 MVP 功能都有对应的页面和用户流程？
- [ ] 每个页面都定义了所有状态（加载/空/正常/错误）？
- [ ] 用户流程覆盖了异常路径？
- [ ] 信息架构的层级合理（重要功能不超过 2 次点击）？
- [ ] 页面的数据需求都列出了（供 API 设计阶段使用）？
- [ ] 没有过度设计（没有 MVP 外的页面）？

## 视觉规范检查
- [ ] 品牌色定义完整（主色+辅色+语义色+背景+文字色）
- [ ] 排版层级定义（至少 H1-H3 + Body + Caption）
- [ ] 圆角风格统一
- [ ] 间距基于统一刻度（不是随机数字）
- [ ] 色值用 CSS 变量表达

## 动效规范检查
- [ ] 动效程度已选择且适合产品类型
- [ ] 过渡时长有标准（不是每个地方不同的随机值）
- [ ] 加载模式已确定（骨架屏/旋转器/渐进式）
- [ ] 尊重 prefers-reduced-motion
