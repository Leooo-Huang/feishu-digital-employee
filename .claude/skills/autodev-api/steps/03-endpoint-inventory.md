# 步骤 3：端点清单

从 UI 操作倒推 API 端点。

**方法：逐页扫描**
1. 打开 UI 文档的每个页面设计
2. 每个页面需要什么数据？-> GET 端点
3. 每个页面有什么操作？-> POST/PUT/DELETE 端点
4. 合并相似端点，避免重复

**格式：**

```markdown
### {模块名}（对应 UI 的 {页面/功能}）
| 方法 | 路径 | 描述 | 对应 UI |
|------|------|------|---------|
| GET | /api/items | 获取列表 | 首页列表 |
| GET | /api/items/:id | 获取详情 | 详情页 |
| POST | /api/items | 创建 | 新建表单 |
| PUT | /api/items/:id | 更新 | 编辑表单 |
| DELETE | /api/items/:id | 删除 | 删除按钮 |
```

<IMPORTANT>
每个端点都要标注「对应 UI」——它服务于哪个页面的什么操作。如果一个端点找不到对应的 UI，要么 UI 设计遗漏了，要么这个端点不该存在。
</IMPORTANT>
