# 步骤 2：数据模型

从 UI 数据需求中提取实体，定义核心数据模型。

**方法：从 UI 页面倒推**
1. 列出所有页面展示的数据字段
2. 将字段归类为实体（如：用户、文章、评论）
3. 定义实体间关系

**格式：**

```
实体：{名称}
描述：{这个实体代表什么}

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| id | string/uuid | 是 | 主键 | 唯一标识 |
| name | string | 是 | 1-100 字符 | 显示名称 |
| created_at | datetime | 是 | 自动生成 | 创建时间 |
| ... | ... | ... | ... | ... |

关系：
- 一个 {A} 有多个 {B}（1:N）
- {A} 和 {C} 多对多（M:N，通过 {关联表}）
```

**设计原则：**
- 只定义 UI 用到的字段，不预设"将来可能需要"的字段
- 主键、时间戳、软删除等通用字段可以统一约定，不用每个实体重复
- 关系要从用户视角验证：用户能看到 A 下面的 B 列表吗？-> 需要 1:N 关系
